一、项目背景与意义

1.1 项目背景

在当前求职与职业发展中，面试是核心环节，但求职者普遍存在面试后复盘难、总结效率低、问题定位不精准的痛点：

人工复盘依赖记忆，容易遗漏关键问题与回答细节，无法全面还原面试场景；

缺乏专业的面试问题拆解、回答优化建议，难以针对性提升面试能力；

批量面试记录管理混乱，Excel 等工具无法实现结构化存储与智能分析。

基于此，我设计并开发了面试 AI 复盘系统，以 FastAPI 为后端框架，集成大语言模型（LLM）与语音识别（ASR）能力，实现面试记录的结构化管理、批量导入与 AI 智能复盘，为求职者提供一站式面试能力提升解决方案。

1.2 项目目标

搭建高可用、易扩展的后端服务，支持面试记录的增删改查全生命周期管理；

集成 ASR 能力，支持语音面试内容自动转文字，还原完整面试对话；

接入 LLM 大模型，实现面试问题拆解、回答质量分析、优化建议生成的智能复盘；

支持 Excel 批量导入面试记录，适配企业 HR 批量管理、求职者多面试场景；

提供全中文友好的 API 文档，降低前后端对接与系统使用门槛。

三、技术原理与核心实现

3.1 核心技术原理

（1）FastAPI 高性能 Web 框架原理

FastAPI 基于 \*\*Starlette（异步 Web 服务器）和Pydantic（数据校验）\*\* 构建，核心优势：

异步支持：原生支持异步 IO，高并发场景下性能远超 Flask/Django；

自动类型校验：通过 Pydantic 模型自动校验请求参数，减少手动校验代码；

自动 API 文档：基于 OpenAPI 标准自动生成 Swagger 文档，本项目二次开发为全中文界面；

类型提示驱动：基于 Python 类型提示实现代码补全、静态检查，提升开发效率。

（2）LLM 大模型复盘原理

系统接入千问等大语言模型，基于prompt 工程实现面试复盘：

输入：完整面试对话（问题 + 回答）、面试岗位、面试场景；

原理：通过精心设计的系统提示词，引导大模型完成「问题拆解→回答评估→优化建议→能力提升」的全流程分析；

输出：结构化的复盘报告，包含问题考点、回答优缺点、优化话术、能力提升方向。

（3）ASR 语音识别原理

集成语音识别 API，实现面试音频自动转文字：

原理：通过预训练的语音识别模型，将音频波形特征转换为文本内容；

作用：自动还原面试对话，无需手动录入，提升复盘效率；

适配：支持常见音频格式（wav、mp3），适配线上面试录音场景。

（4）Excel 批量导入原理

基于openpyxl/pandas实现 Excel 文件解析：

原理：读取 Excel 表格数据，通过 Pydantic 模型校验数据合法性；

流程：文件上传→格式校验→数据解析→批量入库→返回导入结果；

作用：支持 HR 批量导入候选人面试记录，适配企业级使用场景。
